# 我的彩盒结构封板说明

日期：2026-03-23  
状态：高保真前的结构冻结说明

## 1. 目标

这份说明只冻结 `我的彩盒` 的结构、状态关系、页面职责。  
不解决视觉风格、材质、动效细节。

## 2. 最小结构

`我的彩盒` 在 MVP 阶段先收为“两层半”：

1. `页列表`
2. `单页回看`
3. `从单页轻回新的观察`

不增加：

- 分类体系
- 编辑体系
- 管理工具栏
- 复杂筛选

## 3. 页列表

`页列表` 的结构冻结如下：

- 单列叠页
- 最新收进来的页在最上
- 首屏是一张较大的主页片 + 下一页轻露头
- 不是双列卡片
- 不是相册网格
- 不是普通 feed

单张页片的信息顺序冻结如下：

1. `对象图`
2. `色组`
3. `轻线索`

其中：

- `对象图` 是第一眼
- `色组` 为独立色块，视觉成组
- `轻线索` 只承担时间/主题/场景提醒

页列表不承担：

- 颜色名字阅读
- 再次混色
- 解释层内容
- 重操作按钮

## 4. 新收页打开态

`收进彩盒` 后的默认落点冻结为：

- `彩盒 / 新收页打开态`

不回：

- 首页
- 列表页

这个状态的职责是：

- 把用户心智从过程态切入拥有态
- 告诉用户“这一页现在已经属于彩盒了”

## 5. 单页回看

`单页回看` 与 `新收页打开态`：

- 共用同一内容骨架
- 不是同一时态

区别冻结如下：

- `新收页打开态`：强调刚刚收进来的在场感
- `单页回看`：强调稳定地回看这一页

`单页回看` 不承担：

- 编辑旧页
- 重新命名
- 修改来源

## 6. 返回与进入关系

冻结如下：

- `Try -> 收进彩盒 -> 新收页打开态`
- `新收页打开态 -> 页列表`
- `页列表 -> 单页回看`
- `单页回看 -> 再去看看 -> 新的观察`

说明：

- `再去看看` 是重新开始新的一页，不是编辑旧页
- `我的彩盒` 当前不承担旧页编辑职责

## 7. 留待高保真/动效解决的问题

以下问题不在结构层解决：

- `收拢中的中间态` 的真实手感
- `再去看看` 的最终按钮/入口气质
- `我的彩盒一角` 在首页的最终表达
- 页片的材质感、阴影、过渡、玻璃态等视觉语言

## 8. 后续工作边界

高保真阶段只回答：

- 这些结构如何更成立
- 哪种材质与层级最符合 `彩盒`
- 动效如何把“收进来”做对

不再回退讨论：

- 页列表是否双列
- 收完是否回首页
- 单页回看与新收页打开态是否同一状态
