# 彩盒核心体验重定义计划

## 一、问题诊断

### 原始 idea 的核心

用户最初想做的是：

- 帮助孩子认识和感知**自然界的颜色**
- **颜色混合会出什么效果**的探索
- 孩子自然而然尝试的辅助**玩具**

### 设计过程中的偏移

在设计演进中，产品逐渐变成了：

- "颜色关系工作台"
- "看清主次与托举"
- "颜色作用实验台"

这些表达越来越"设计师化"，离孩子最自然的好奇心越来越远。**混色**——这个最初就被提到的核心体验——被抽象化为"关系"，丢失了孩子能理解、能惊喜的可玩性。

## 二、新的产品定义

**彩盒**：帮助孩子从自然观察中发现颜色，通过混色实验理解颜色的秘密，并把这些发现收成一个个小故事。

### 核心体验循环

```
观察自然 → 发现多种颜色 → 混色实验（惊喜！）→ 收成一个故事
```

这是一个"科学发现"循环：

1. **观察**：这片叶子好看
2. **发现**：原来它不只有一种绿，还有黄、深绿、浅绿
3. **实验**：如果我把这个黄和这个蓝混在一起...
4. **惊喜**：哇，混出来一个新颜色！
5. **记录**：这是我发现这片叶子秘密的故事

### 与现有混色 App 的区别

市面上的混色 App 是：抽象的 A + B = C

彩盒的独特之处：混色有**来源**，有**故事**

- 颜色是从真实观察中带出来的
- 混色是在探索"这些颜色是怎么长出来的"
- 结果不只是新颜色，而是"我的发现"

## 三、重新定义主路径

### 当前路径（存在问题）

```
Home → See/Live → See/Frozen → See/Picked → 显色(?) → 收进彩盒 → 我的彩盒
```

问题在于"显色"始终没有找到让孩子能理解、能玩、能惊喜的形态。

### 新路径

```
Home → 观察(Live) → 发现(Frozen) → 混色实验 → 收成故事 → 我的故事册
```

### 各页职责重新定义

**Home**：基本保持现状，调整入口表达，从"去看看今天的颜色"到更接近"今天想探索什么"

**观察(Live)**：把这一刻停住，保持现状

**发现(Frozen)**：

- 核心任务不变：指出"我注意到哪里"，带出颜色
- 微调：强调"发现这里面有好几种颜色"的惊喜

**混色实验**（原"显色"，重大重定义）：

- 核心交互：从发现的颜色中选择，混合，看会变成什么
- 惊喜感：混出新颜色的那一刻
- 探索空间：多试几次，比例怎么影响结果
- 可选连接：混出来的颜色和图上某处像不像

**收成故事**（原"收进彩盒"，重新定义）：

- 不是保存颜色卡片
- 而是收成一个小故事：我看到了什么 → 发现了什么 → 试了什么

**我的故事册**（原"我的彩盒"，重新定义）：

- 不是颜色档案库
- 而是故事册：每一页是一个发现的故事

## 四、需要解决的核心设计问题

### 1. 混色交互怎么做？

需要回答：

- 从已发现的颜色中选两个：怎么选？点击？拖拽？
- 混合的动作是什么：拖到一起？按按钮？
- 比例怎么调：滑杆？还是简化为"多一点/少一点"？
- 混合结果怎么出现：有惊喜感的过渡

建议方向：

- 简单直接：点选两个颜色 → 看到它们混合 → 新颜色出现
- 比例可作为进阶能力，初期可简化

### 2. 混色结果怎么呈现？

需要回答：

- 新颜色出现时的"惊喜感"怎么做
- 多次混色的结果怎么累积
- 和来源图的关系怎么保持（视觉上）

建议方向：

- 来源图始终在场，但作为背景/参照
- 每次混出新颜色，是一个"发现时刻"
- 可以尝试多次混色，形成一组"实验记录"

### 3. 什么时候算"完成"？

需要回答：

- 自由探索，随时可收？
- 还是有明确的"完成"状态？

建议方向：

- 保持"随时可收"的自由度
- 但给出"这一次探索可以收了"的自然提示
- 不强制完成感，符合原有设计原则

### 4. 故事页长什么样？

需要回答：

- 来源图 + 发现的颜色 + 混色过程 + 结果
- 如何让它像一个"故事"而不是"报告"

建议方向：

- 以来源图为封面
- 展示发现的颜色 + 混色尝试 + 结果
- 可以有极轻的叙事提示（"我发现..."、"我试了..."）
- 时间感/场景感作为轻线索

## 五、已确定可保留的部分

基于 [我的彩盒结构封板说明](docs/plans/2026-03-23-我的彩盒结构封板说明.md)：

- **页列表的结构**：单列叠页，最新在上
- **单页回看的骨架**：对象图 + 核心内容 + 轻线索
- **收完后的落点**：新收页打开态，不回首页

但这些结构的**内容定义**需要调整，从"颜色档案"变为"发现故事"。

- 这是当前最大的空白，也是新方向的核心
- 需要先回答混色交互的基本形态
- 建议先用最简单的方式验证：选两个颜色 → 混合 → 看结果

### 第三步：回过头调整前段

- Frozen 的微调：强调"发现多种颜色"
- Home 入口文案的微调

- 基于新定义，重做 prototype-v2 中的关键页

## 七、仍需讨论的开放问题

1. **混色的物理感**：是像颜料混合，还是像光混合，还是更抽象？
2. **AI 的角色**：在混色实验中，AI 是否需要出场？以什么方式？
3. **与"比较"的关系**：用户选择的三个核心画面之一是"跨时间比较"，这个能力在新路径中如何体现？
