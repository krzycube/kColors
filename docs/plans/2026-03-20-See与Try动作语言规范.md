# See 与 Try 动作语言规范

日期：2026-03-20

用途：统一 `See` 与 `Try` 页的顶部导航、状态提示、底部动作的命名和职责，避免页面文案混杂不同层级的行为语义。

## 一、核心原则

### 1. 顶部栏只承担两类职责

- 导航
- 极轻的上下文提示

顶部栏不承担：

- 流程推进
- 页面收束
- 结果提交

因此，不应把 `继续`、`收起` 这类流程或容器语义放在顶部栏。

### 2. 底部动作区承担状态推进

底部动作区用于表达：

- 当前状态下还能做什么
- 下一步最自然的动作是什么

因此，主次动作应该放在底部，不与顶部导航混用。

### 3. 文案要描述孩子的下一步行为，而不是产品内部动作

优先使用：

- `再找一处`
- `放在一起看看`
- `回到观察`
- `收进彩盒`

避免使用：

- `继续`
- `收起`
- `试试`
- `重新看看`

原因：

- 太空泛
- 太像工具操作
- 太像容器状态
- 不能自然映射到孩子的下一步行为

## 二、页面级规则

## 2.1 See - Initial

顶部：

- 左：`返回`
- 中：`点一点你看到的颜色`
- 右：`相册` 图标入口

底部：

- 保留与后续状态一致的交互轨道空间
- 但不提前亮出完整主次动作按钮
- 只呈现安静的保留位

意图：

- 让孩子先观察
- 但页面结构不因后续状态切换而跳动

## 2.2 See - 3 Colors

顶部：

- 左：`返回`
- 中：`原来这里不止一种绿色`
- 右：默认不放流程动作；是否保留相册入口可后续再定

底部：

- 不再显式提供次动作按钮
- 主动作：`放在一起看看`

意图：

- 用户此时仍然天然处在观察空间内，不点主动作即可继续观察
- 主动作自然过渡到 `Try`

## 2.3 Try

顶部：

- 左：`回到观察`
- 中：来源 chip，例如 `这些颜色来自刚才那片叶子`
- 右：无额外动作

底部：

- 不再重复提供次动作按钮
- 主动作：`收进彩盒`

意图：

- 顶部表达来源连续感
- 底部只承担最终收纳
- 不使用 `收起` 这类容器语义

## 三、结构约束

### 1. See 的两个状态本质上是同一页

因此：

- 对象区高度一致
- 底部交互轨道高度一致
- 状态切换时只改变轨道内部内容

### 2. Try 的动作区是稳定的底部操作簇

因此：

- `结果卡 + 按钮区` 作为一组稳定存在
- 靠近底部安全区，但保留固定呼吸距离
- 不随着上方内容高度随意漂移

## 四、后续执行要求

- 先按本规范统一 `Pencil` 当前 `See / Try`
- 后续无论继续在 `Pencil` 还是吸收 `Calicat` 稿件，都以本规范为准
- 如果将来新增图标化导航，也不能改变顶部与底部的职责边界
