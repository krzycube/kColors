# prototype-v2 更新记录

日期：2026-03-27  
背景：基于[彩盒核心体验重定义计划](.cursor/plans/彩盒核心体验重定义_882e5c5e.plan.md)执行原型更新

## 一、核心方向变更

### 从"显色"到"混色实验"

原设计方向：
- "颜色关系工作台"
- "看清主次与托举"
- "颜色作用实验台"

新设计方向：
- "混色实验" — 选两个颜色，混在一起，看会变成什么
- 核心体验：**惊喜感** — 混出新颜色的那一刻

### 从"彩盒"到"故事册"

原设计方向：
- 颜色档案库
- 颜色卡片的堆叠

新设计方向：
- 故事册 — 每一页是一个发现的故事
- 叙事结构："我发现了..." → "我试着混了混..." → "我得到了..."

## 二、页面修改清单

### 1. 混色实验页（原"显色"页）

**文件变更：**
- `index.html`: 重写了 `data-screen="xianse"` section
- `styles/screens/mix.css`: 新增混色实验页样式
- `app.js`: 新增混色状态管理和交互逻辑

**页面结构：**
```
顶部导航：返回 | 来源chip | 收成按钮
标题区：混色实验 / 从下面选两个颜色...
实验台：左槽位 + 中间结果 + 右槽位
颜色选择区：发现的颜色 | 混出的新颜色
实验记录：混色历史列表
底部CTA：收成这次发现
```

**交互流程：**
1. 点击"发现的颜色"中的某个颜色
2. 颜色飘到实验台左侧槽位
3. 再点击另一个颜色
4. 颜色飘到右侧槽位
5. 自动混合（400ms 延迟后触发）
6. 新颜色出现在中间结果区（带动画）
7. 新颜色加入"混出的新颜色"区域
8. 可继续实验或收成

**混色算法：**
- MVP 阶段使用简单的 RGB 平均值混合
- `mixColors(color1, color2)` → 两色 RGB 各取平均

## 三、交互引导优化（用户反馈后迭代）

### 问题
用户测试发现：进入混色实验页后，会在实验台的"选一个/再选一个"槽位上点击，不知道要点击下方的颜色列表。

### 解决方案

1. **标题引导强化**
   - 从"选两个颜色看看会变成什么"
   - 改为"从下面选两个颜色，看看混在一起会怎样"

2. **颜色选择区引导**
   - 标签从"发现的颜色"
   - 改为"点击颜色开始混色 ↓"

3. **实验台槽位弱化**
   - 占位符从"选一个/再选一个"
   - 改为"…"（省略号）
   - 视觉上更淡（opacity: 0.3），看起来是等待状态而非按钮

4. **颜色按钮脉冲动画**
   - 新增 `colorPulse` 动画
   - 颜色圆点轻微呼吸，暗示可点击
   - hover 时动画停止，放大效果
