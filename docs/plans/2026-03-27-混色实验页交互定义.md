# 混色实验页交互定义

日期：2026-03-27  
状态：核心交互定义稿  
用途：作为混色实验页设计与实现的执行依据

## 一、页面定位

### 工作名
`混色实验` 或 `调色`

### 核心命题
让孩子把从自然中发现的颜色混合在一起，看看会变成什么新颜色。

### 它不是什么
- 不是配色工具
- 不是颜色关系分析页
- 不是修图器
- 不是颜料模拟器

### 它是什么
- 是一个让孩子亲手试"A + B = ?" 的实验台
- 是一个产生惊喜的地方
- 是观察与记录之间的桥梁

## 二、进入条件

从 `Frozen / 已选态` 进入，携带：
- 来源图
- 2-5 个已发现的颜色片段

## 三、页面结构

### 三层结构

```
┌─────────────────────────────────┐
│         顶部导航层              │
│   返回  |  来源缩略  |  收成    │
├─────────────────────────────────┤
│                                 │
│         混色实验台              │
│    （页面核心，占主要空间）     │
│                                 │
├─────────────────────────────────┤
│         颜色选择区              │
│   已发现的颜色 + 混出的新颜色   │
└─────────────────────────────────┘
```

### A. 顶部导航层

职责：
- 返回（回到 Frozen）
- 来源连续感（来源图的极小缩略或 chip）
- 收成入口（当有混色结果时）

不承担：
- 复杂说明
- 工具面板

### B. 混色实验台（核心区域）

这是页面的主舞台，承载混色的核心交互。

#### 视觉呈现
- 一个干净的"实验区域"
- 背景可带有来源图的极淡映照，保持来源连续感
- 中心是混色发生的地方

#### 状态
- 空态：等待选择两个颜色
- 选中一个：显示第一个颜色，等待第二个
- 选中两个：准备混合
- 混合中：过渡动画
- 混合完成：新颜色出现，惊喜时刻

### C. 颜色选择区

职责：
- 展示从 Frozen 带来的颜色
- 展示已经混出的新颜色
- 提供选择交互

布局：
- 水平排列
- 已发现的颜色在左侧
- 混出的新颜色在右侧（或下方新区域）

## 四、核心交互流程

### Step 1: 选择第一个颜色

交互方式：点击颜色选择区中的某个颜色

反馈：
- 该颜色被选中，有视觉高亮
- 该颜色"飘"到实验台的左侧位置
- 提示："再选一个"

### Step 2: 选择第二个颜色

交互方式：点击另一个颜色

反馈：
- 该颜色被选中
- 该颜色"飘"到实验台的右侧位置
- 两个颜色并排呈现，中间有"混合"的暗示

### Step 3: 触发混合

交互方式选项（推荐方案 A）：

**方案 A：自动混合**
- 当两个颜色都选中后，自动开始混合动画
- 不需要额外按钮
- 更流畅，更有惊喜感

**方案 B：手动触发**
- 选中两个后，点击中间的"混合"按钮
- 更有控制感，但多一步操作

**方案 C：拖拽合并**
- 把一个颜色拖到另一个上面
- 更直觉，但对儿童可能有操作难度

当前推荐：方案 A（自动混合）

### Step 4: 混合动画

动画方向：
- 两个颜色向中间靠拢
- 在中间"融合"
- 新颜色从融合点"绽放"出来

动画时长：约 800ms-1200ms

惊喜感设计：
- 融合过程有轻微的"膨胀"感
- 新颜色出现时有轻微的"弹出"或"发光"
- 不要太花哨，保持温柔气质

### Step 5: 看到新颜色

新颜色的呈现：
- 停留在实验台中央
- 大小比选择区的颜色更大（强调它是"结果"）
- 可以有极轻的命名提示（可选，如"这是一种暖绿"）

同时：
- 新颜色被复制到颜色选择区
- 可以继续用它参与下一次混色

### Step 6: 继续实验 / 收成故事

继续实验：
- 点击选择区的任意颜色，开始新一轮
- 可以用原始颜色，也可以用刚混出的新颜色
- 形成"混色链"

收成故事：
- 当觉得探索够了，点击顶部"收成"
- 进入故事收成流程

## 五、颜色选择区详细设计

### 布局

```
已发现的颜色        混出的新颜色
[●][●][●][●][●]  |  [●][●][●]...
```

### 每个颜色的呈现

- 圆形色块（直径约 44pt，符合点击热区）
- 当前选中的颜色有边框高亮
- 已参与当前混色的颜色有轻微变暗或标记

### 区分原始颜色与混出颜色

视觉区分：
- 原始颜色：保留来源痕迹（可选：极小的来源裁片角标）
- 混出颜色：有"新"的感觉（可选：轻微光晕或渐变边缘）

或者更简单：用位置区分，不加复杂标记

### 数量限制

- 原始颜色：2-5 个（从 Frozen 带来）
- 混出颜色：建议最多 5-8 个，避免过度堆积
- 如果超出，最早的混出颜色可以淡出或提示"收成后继续"

## 六、特殊情况处理

### 同一颜色混两次？

允许，结果就是原颜色（或可以不允许选同一个两次）

当前推荐：不允许选同一个两次，给出轻提示

### 想撤销选择？

再次点击已选中的颜色，取消选中

### 想清空实验台？

点击实验台空白处，或提供轻量的"重选"入口

### 混色比例？

MVP 阶段：固定 1:1 混合，简化交互

未来可扩展：
- "多一点这个"的简单调节
- 或滑杆调比例（进阶能力）

## 七、与来源图的关系

### 来源图如何在场？

选项：
1. 顶部小缩略（chip 形式）
2. 实验台背景极淡映照
3. 混色结果旁的来源对比入口

当前推荐：顶部小缩略 + 实验台背景极淡映照

### 验证功能（可选）

"这个混出来的颜色，和图上的哪里像？"

当前判断：这是 nice to have，不是 MVP 必须
可以作为后续版本的扩展

## 八、收成触发

### 什么时候可以收成？

- 只要混出过至少 1 个新颜色
- 或者即使没混色，只是看了看（允许轻量故事）

### 收成入口

- 位置：顶部导航层右侧
- 状态：
  - 无混色结果时：隐藏或极弱化
  - 有混色结果时：可见但不强推
- 文案方向："收成这次发现" 或简单的图标

## 九、页面状态总结

| 状态 | 实验台内容 | 选择区状态 | 收成入口 |
|------|-----------|-----------|---------|
| 初始 | 空，等待选择 | 原始颜色可选 | 隐藏/弱化 |
| 选中1个 | 显示第一个颜色 | 第一个高亮，其他可选 | 隐藏/弱化 |
| 选中2个 | 两个颜色并排 | 两个高亮 | 隐藏/弱化 |
| 混合中 | 动画进行中 | 不可操作 | 隐藏 |
| 混合完成 | 新颜色呈现 | 新颜色加入，全部可选 | 可见 |
| 多次混色后 | 最近一次结果 | 原始+所有混出颜色 | 可见 |

## 十、动效要点

### 颜色选中
- 颜色轻轻"跳起"或"发光"
- 然后平滑移动到实验台

### 混合过程
- 两色靠拢
- 边缘开始互相渗透
- 融合点膨胀
- 新颜色"绽放"

### 新颜色加入选择区
- 从实验台"复制"一份
- 飘到选择区的混出颜色位置
- 轻轻落下

### 整体节奏
- 不急促
- 每个动作之间有呼吸
- 让孩子有时间"哇"

## 十一、与已有设计的关系

### 继承的原则
- 系统越来越隐身，发现越来越显眼
- 随时可停，随时可回来
- 不强制完成感

### 替代的设计
- 原"显色"的"主次关系""托举""站位"等概念
- 原"颜色作用实验台"的复杂方向

### 保留的结构
- 顶部导航的基本布局
- 收成后进入"新收页打开态"
- 最终进入"我的彩盒/故事册"
