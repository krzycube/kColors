# 故事页内容结构定义

日期：2026-03-27  
状态：内容结构定义稿  
用途：定义"收成一页"后形成的故事页，作为设计与实现的执行依据

## 一、核心定位

### 它是什么

每一页故事是孩子的一次"发现之旅"的记录：

> 我看到了什么 → 发现了什么颜色 → 试了什么混色 → 得到了什么

### 它不是什么

- 不是颜色卡片的堆叠
- 不是数据报告
- 不是相册
- 不是配色方案展示

### 故事感的来源

故事感不来自大段文字，而来自：
1. **时间顺序**：有开始、过程、结果
2. **个人视角**：是"我"的发现，不是客观描述
3. **因果关系**：因为看到了这个，所以发现了那个，然后试了这个

## 二、故事页的三个版本

根据使用场景，故事页有三种呈现形态：

### 1. 新收页打开态

从混色实验"收成"后的第一眼

职责：
- 确认"这一页被收住了"
- 建立拥有感
- 保留"再去看看"的轻热度

特点：
- 强调刚刚发生的在场感
- 可能有轻微的"刚收好"的视觉提示

### 2. 单页回看态

从故事册中点开某一页

职责：
- 回顾这次发现
- 重新体验那个时刻

特点：
- 更稳定、更完整
- 不再强调"刚刚收好"

### 3. 页列表中的缩略态

在故事册列表中的呈现

职责：
- 快速识别这是哪次发现
- 邀请点进去看

特点：
- 信息精简
- 以来源图为主视觉

## 三、单页完整结构

### 信息层级（从上到下）

```
┌─────────────────────────────────┐
│           封面区                │
│     来源图 + 时间/场景线索      │
├─────────────────────────────────┤
│           发现区                │
│    "我发现了这些颜色"           │
│    [●][●][●][●]                 │
├─────────────────────────────────┤
│           实验区                │
│    "我试着把它们混在一起"       │
│    混色过程的简化呈现            │
├─────────────────────────────────┤
│           结果区                │
│    "我得到了这些新颜色"         │
│    [●][●]                       │
├─────────────────────────────────┤
│           轻动作区              │
│    再去看看 / 分享 / 其他       │
└─────────────────────────────────┘
```

### A. 封面区

#### 内容
- **来源图**：占据主要视觉，是这个故事的"封面"
- **时间线索**：极轻的时间标记，如"今天下午"或"3月27日"
- **场景线索**（可选）：如果能识别，可以有"一片叶子"之类的轻提示

#### 视觉
- 来源图不必完整展示，可以是关键区域的裁切
- 图片上可以有已标记的"发现点"的轻痕迹
- 整体氛围温暖、安静

#### 尺寸
- 高度约占页面 40-50%
- 不要太小，它是故事的"场景"

### B. 发现区

#### 内容
- 轻叙事提示："我发现了这些颜色" 或更简单的 "发现"
- 从来源中提取的原始颜色（2-5个）

#### 视觉
- 颜色以圆形色块呈现，水平排列
- 每个颜色可以保留极轻的来源关联（如细线指向封面图的位置）
- 也可以不做关联，保持简洁

#### 叙事语气
- 第一人称
- 简短
- 不解释"为什么发现"，只呈现"发现了什么"

### C. 实验区

#### 内容
- 轻叙事提示："我试着混了混" 或 "实验"
- 混色过程的简化呈现

#### 混色过程的呈现方式

**方案 A：公式式**
```
● + ● = ●
绿   黄   浅绿
```
简洁、清晰、像科学记录

**方案 B：流程式**
```
● ──→ ● ──→ ●
     混合
```
更有过程感

**方案 C：堆叠式**
```
    ●
   / \
  ●   ●
```
更有视觉感，但可能复杂

当前推荐：方案 A（公式式），简洁直观，孩子能理解

#### 多次混色怎么呈现？
- 如果混了多次，垂直堆叠多个公式
- 最多展示 3-4 条，更多的折叠
- 或者只展示"最有意思的"几次（系统判断或用户选择）

### D. 结果区

#### 内容
- 轻叙事提示："我得到了" 或 "新颜色"
- 混出的新颜色（1-若干个）

#### 视觉
- 新颜色以稍大的圆形色块呈现
- 可以有轻微的"新"的感觉（如微光或渐变边缘）
- 如果有多个新颜色，水平排列

#### 特殊情况
- 如果没有混色，只是观察，这一区可以省略或变为"这次我只是看了看"

### E. 轻动作区

#### 内容
- "再去看看"：开始新的观察
- 分享（可选，后续版本）
- 其他轻动作

#### 视觉
- 不抢眼
- 在页面最底部
- 文字链或极轻的按钮

## 四、叙事语言设计

### 原则
- 第一人称
- 短句
- 不解释，只呈现
- 温柔但不幼稚

### 叙事片段示例

封面区：
- （不需要文字，图片即叙事）
- 或极轻的场景词："一片叶子"

发现区：
- "我发现了这些颜色"
- 或更简短："发现"
- 或无文字，纯图形

实验区：
- "我试着混了混"
- "把这两个放在一起会怎样？"
- 或更简短："实验"

结果区：
- "我得到了"
- "新颜色"
- 或无文字，只有色块

### 文字可选策略
- MVP 阶段可以用极简文字
- 后续可以根据孩子年龄调整：
  - 更小的孩子：纯图形
  - 更大的孩子：可以自己写备注

## 五、新收页打开态的特殊处理

### 与普通回看的区别

| 维度 | 新收页打开态 | 单页回看态 |
|------|------------|-----------|
| 时态 | 刚刚发生 | 回忆 |
| 视觉 | 可能有"刚收好"的提示 | 稳定呈现 |
| 动作 | 强调"再去看看" | 更平衡的动作 |
| 情绪 | 拥有感 + 轻热度 | 回味 |

### "刚收好"的视觉提示选项
- 页面边缘有轻微的"新"的光晕
- 或顶部有一行极轻的"这一页收好了"
- 或无特殊处理，只靠过渡动效传达

当前推荐：靠过渡动效传达，不加显式文案

## 六、页列表中的缩略呈现

### 单张页片的内容

```
┌─────────────────────────────────┐
│                                 │
│         来源图（主视觉）         │
│                                 │
├─────────────────────────────────┤
│  [●][●][●]     今天下午         │
│  发现的颜色    时间线索          │
└─────────────────────────────────┘
```

### 信息优先级
1. 来源图 — 第一眼识别这是哪次发现
2. 发现的颜色 — 快速看到核心内容
3. 时间线索 — 知道是什么时候

### 不需要在缩略中呈现
- 混色过程
- 新颜色结果
- 叙事文字

这些留给点进去后的完整页面

## 七、数据结构（供实现参考）

### 一页故事包含

```javascript
{
  id: "story-001",
  createdAt: "2026-03-27T15:30:00",
  
  // 来源
  source: {
    type: "live" | "import" | "sample",
    imageUrl: "...",
    thumbnail: "..."
  },
  
  // 发现的颜色
  discoveredColors: [
    { id: "c1", hex: "#4a7c59", position: {x, y} },
    { id: "c2", hex: "#8fb339", position: {x, y} },
    ...
  ],
  
  // 混色记录
  mixingExperiments: [
    {
      inputs: ["c1", "c2"],
      ratio: [1, 1],  // MVP 固定 1:1
      result: { id: "m1", hex: "#6a9a49" }
    },
    ...
  ],
  
  // 混出的新颜色
  createdColors: [
    { id: "m1", hex: "#6a9a49" },
    ...
  ],
  
  // 轻线索（可选）
  hints: {
    scene: "一片叶子",  // AI 识别或用户输入
    timeLabel: "今天下午"
  }
}
```

## 八、与已有设计的关系

### 继承的结构
基于 [我的彩盒结构封板说明](docs/plans/2026-03-23-我的彩盒结构封板说明.md)：
- 页列表：单列叠页，最新在上
- 单页回看骨架：对象图 + 核心内容 + 轻线索
- 收完后落点：新收页打开态

### 改变的内容
- 从"颜色档案"变为"发现故事"
- 增加混色过程的呈现
- 增加叙事语言

### 保留的原则
- 不伪造拥有感
- 随时可停，随时可回来
- 系统隐身，发现显眼

## 九、开放问题

1. **混色过程要不要可交互？**
   - 点击混色公式能不能重新看动画？
   - 当前建议：MVP 不做，静态呈现

2. **能不能编辑？**
   - 能不能删除某次混色记录？
   - 当前建议：MVP 不做编辑，保持简单

3. **场景识别**
   - 来源图能不能自动识别"这是叶子"？
   - 当前建议：可选功能，不影响核心体验
