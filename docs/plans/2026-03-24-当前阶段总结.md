# 彩盒当前阶段总结

日期：2026-03-24  
状态：主线整理稿  
用途：作为当前继续推进设计与原型的最小上下文，避免再次被工具协作记录和中间过程文档干扰。

## 一、产品感觉

`彩盒` 不是儿童教育工具，不是颜色教具，不是功能型相机 App。

它是一个帮助孩子独自观察、带回、试着理解颜色关系，并把一次发现收成一页的产品。

它的感觉应当是：

- 温柔，但不软弱
- 安静，但不空
- 有引导，但不替孩子看完
- 有技术能力，但系统应尽量隐身

最核心的设计原则仍然是：

- 先帮孩子发现，而不是立刻解释一切
- 让孩子自己看到更多，而不是替孩子看完
- 系统越来越隐身，发现越来越显眼
- 不能伪造拥有感

## 二、核心体验主线

当前已经确认的最小闭环是：

`Home -> See / Live -> See / Frozen -> See / Picked -> Try -> 收进彩盒 -> 彩盒 / 新收页打开态 -> 我的彩盒 / 页列表 -> 单页回看`

这条主线不是为连续快速采集优化，而是为：

`一次观察，形成一页发现`

优化。

也就是说，它更鼓励：

- 停一下
- 看一会儿
- 带回几种颜色
- 让它们待一会儿
- 再把这一页收起来

## 三、页面角色

### 1. Home

首页不是功能大厅，而是状态承接页。

首页的职责优先级：

1. 开始新的观察
2. 回到未完成的现场
3. 从精选样本开始
4. 从相册带进来
5. 轻量露出 `我的彩盒`

首页的关键要求：

- 首屏先服务“开始”和“续接”
- `我的彩盒` 不应抢首页主位
- 浮动底部导航可存在，但不能压过主入口

### 2. See

`See` 不是一张页面，而是同一观察空间里的三个连续状态：

- `Live`
- `Frozen`
- `Picked`

`Live` 的任务是把 moment 定住，  
`Frozen` 的任务是开始带回颜色，  
`Picked` 的任务是进入“这些颜色可以放在一起看看”的状态。

### 3. Try

`Try` 不是投放器，不是教具，不是结果解释页。

它是颜色关系工作台。  
它的任务是让已经带回来的颜色彼此待一会儿，再决定是否收进彩盒。

### 4. 我的彩盒

`彩盒` 是容器，`页` 是内容单位。

`我的彩盒` 当前最小结构是“两层半”：

1. `页列表`
2. `单页回看`
3. 从单页轻回新的观察

当前不让 `我的彩盒` 承担旧页编辑。

## 四、已经定住的结构规则

### 1. See 与 Try 的动作边界

- 顶部只承担导航和极轻上下文
- 底部承担状态推进
- 不再使用 `继续`、`收起` 这类空泛动作词

### 2. 收进彩盒之后

点击 `收进彩盒` 后：

- 不回首页
- 不直接落列表
- 默认进入 `新收页打开态`

它的作用是：

- 把用户从过程态切入拥有态
- 让用户感到“这一页现在已经属于我了”
- 但仍然留下轻量的“可以再去看看”的热度

### 3. 我的彩盒页列表

页列表当前冻结为：

- 单列
- 最新页在上
- 首屏为一张主位页片，加轻量后续页提示
- 不做双列卡片
- 不做相册网格

单张页片的内容顺序：

1. 对象图
2. 色组
3. 轻线索

### 4. 首页中的“我的彩盒”

首页里的 `我的彩盒` 只是入口，不是列表本体。

它应该让人感觉：

- 下面压着一页页被收起来的发现

但不应该让首页变成“先回顾，再开始”。

## 五、当前原型工作的方式

当前主设计现场是：

- `/Users/yuxihe/repo/arrwuu/kColors/prototype/index.html`
- `/Users/yuxihe/repo/arrwuu/kColors/prototype/styles.css`
- `/Users/yuxihe/repo/arrwuu/kColors/prototype/app.js`

当前不再把 Pencil 或 Calicat 作为主设计现场。

原因很直接：

- Web 原型更适合同时控制页面结构、状态差异、真实交互和样式层
- 更能验证产品体验，而不是只验证静态画面

## 六、当前最需要继续的工作

### 1. 继续收首页

首页已经比前几轮稳定，但还需要继续统一：

- 首页各区块之间的垂直节奏
- 主入口卡的体量和入口语言
- `上次停在这里` 状态标记的轻重
- 首页中的 `我的彩盒` 入口质感

### 2. 继续补后半段完成度

当前明显落后于首页和 `See` 的，是：

- `Try`
- `新收页打开态`
- `页列表`
- `单页回看`

它们现在已有结构，但还没完全长出属于 `彩盒` 的辨识度。

### 3. 持续减轻解释性文字

对 7 岁孩子来说，当前部分页面仍然有文字偏多的问题。

后续方向：

- 能用状态表达的，不用句子解释
- 能用位置和关系表达的，不用说明性文案补
- 只保留最必要、最短的引导词

## 七、继续推进时的工作约束

后续工作中，不再回退讨论：

- 产品是不是颜色教育工具
- 主线是不是连续快速采集
- See 是否需要 Live / Frozen / Picked
- 收进彩盒后是否回首页
- 我的彩盒是否先做复杂管理和编辑

后续工作只解决：

- 页面完成度
- 页面间的状态差异
- 视觉语言的一致性
- 真机原型前的交互与层级质量
